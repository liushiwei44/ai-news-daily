对所有使用 Gemini API 的开发者而言，今天（5 月 26 日）是一个需要关注的节点：Gemini API v1beta Interactions API 的破坏性变更（Breaking Changes）自今日起正式将新架构切换为默认行为。此次变更重构了 API 的请求/响应 Schema，以支持未来的关键新能力：mid-flight steering（中途修正正在执行的任务）和异步工具调用（Asynchronous Tool Calls，允许多个工具并行执行而不阻塞主流程）。开发者可在 6 月 8 日前通过请求头字段临时选择退出（opt-out）旧 Schema，但 6 月 8 日之后旧 Schema 将被永久移除，届时所有未完成迁移的应用将出现兼容性错误。主要迁移点包括：`contents` 字段结构调整；`tool_calls` 由同步响应改为 `pending_tool_calls` + `tool_results` 异步回调模型；`finish_reason` 枚举值新增 `TOOL_AWAITING` 状态。Google 为此在官方文档中提供了完整的迁移指南和 Colab Notebook 示例。对于在 Anthropic MCP 或 OpenAI Function Calling 下已采用异步工具调用模式的开发者，这次迁移实际上是向行业最佳实践的靠拢，迁移后的 Gemini API 与其他前沿 LLM 提供商的架构一致性将显著提升。
